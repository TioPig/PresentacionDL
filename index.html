1. Introducción

Este proyecto implementa una Red Neuronal Artificial (MLP) para predecir si un cliente bancario utilizará servicios bancarios online (si será "Internauta"). El análisis se basa en características demográficas y financieras de los clientes, con el objetivo de identificar patrones y factores determinantes que influyen en la adopción de servicios digitales.

Objetivos del Proyecto

  • Desarrollar un modelo predictivo preciso para identificar potenciales usuarios de servicios bancarios online
  • Identificar los factores clave que influyen en la adopción de servicios digitales
  • Proporcionar recomendaciones estratégicas para aumentar la digitalización de clientes

La capacidad de predecir qué clientes tienen mayor probabilidad de utilizar servicios bancarios online permite a las instituciones financieras optimizar sus estrategias de marketing digital, mejorar la experiencia del usuario y reducir costos operativos asociados a canales tradicionales.

3. Preprocesamiento de Datos

Para preparar los datos para el entrenamiento del modelo, realizamos varias transformaciones:

3.1 Variables Seleccionadas

  • Variables Numéricas: Renta, Edad, Antigüedad
  • Variables Categóricas: Región, Sexo
  • Variable Objetivo: Internauta (binaria: 0/1)

3.2 Transformaciones Aplicadas

  • Estandarización de variables numéricas (media 0, desviación estándar 1)
  • Codificación One-Hot para variables categóricas
  • División de datos: 70% entrenamiento, 15% validación, 15% prueba

Importancia del Preprocesamiento

La estandarización de variables numéricas es crucial para el entrenamiento eficiente de redes neuronales, ya que ayuda a que el algoritmo de optimización converja más rápidamente. La codificación One-Hot permite que el modelo interprete correctamente las variables categóricas sin imponer una relación ordinal artificial.

4. Implementación del Modelo

Implementamos una Red Neuronal Artificial (MLP) utilizando PyTorch, con capacidad para personalizar la arquitectura, funciones de activación y técnicas de regularización.

Resultado de la Experimentación

El "Modelo con Dropout" mostró el mejor rendimiento general, con un F1-score de 0.8341 y una exactitud de 0.7453. Este modelo logra un buen equilibrio entre precisión y recall, siendo especialmente efectivo en la detección de la clase positiva (internautas).

5. Análisis de Funciones de Activación

Realizamos un experimento adicional para comparar el rendimiento de diferentes funciones de activación: ReLU, Tanh, Sigmoid, ELU y SELU.

Comparación de métricas entre diferentes funciones de activación

Hallazgos sobre Funciones de Activación

La función de activación ELU (Exponential Linear Unit) obtuvo el mejor F1-score (0.8322) y la mejor exactitud (0.7442). Esta función combina las ventajas de ReLU (rápida convergencia) con una transición suave para valores negativos, lo que ayuda a mitigar el problema de "neuronas muertas".

La función Sigmoid mostró un comportamiento patológico, clasificando todos los ejemplos como positivos, lo que resultó en un AUC-ROC de 0.5 (equivalente a una predicción aleatoria). Esto confirma que Sigmoid no es adecuada para capas ocultas en este problema.

6. Efecto del Tamaño de Batch

Analizamos cómo el tamaño de batch afecta tanto el rendimiento del modelo como el tiempo de entrenamiento, probando tamaños de 16, 32, 64, 128 y 256.

Comparación de métricas por tamaño de batch
Tiempo de entrenamiento por tamaño de batch

Tamaño de Batch Óptimo

El tamaño de batch 128 ofrece el mejor equilibrio entre rendimiento predictivo (F1-score de 0.8308) y eficiencia computacional. Los tamaños más pequeños (16, 32) aumentan significativamente el tiempo de entrenamiento sin mejorar el rendimiento, mientras que el tamaño 256 reduce el tiempo pero con una ligera disminución en las métricas.

Importancia de características basada en pesos de la primera capa

Factor Determinante: Edad

Ambos métodos confirman que la edad es, por un amplio margen, la característica más importante para predecir si un cliente será internauta. En el análisis de permutación, la edad tiene una importancia 42 veces mayor que la segunda característica más importante (Renta).

La renta tiene una importancia moderada pero significativamente menor que la edad. Algunas regiones muestran una pequeña influencia en la predicción, mientras que la antigüedad como cliente tiene un impacto muy limitado.

9. Predicciones para Diferentes Perfiles

Aplicamos el modelo a diferentes perfiles de clientes para ilustrar su capacidad predictiva y entender mejor los patrones identificados.

Predicciones del modelo para diferentes perfiles de clientes
Probabilidad de ser internauta por categoría de edad y renta

Patrones Identificados

Se confirma que la edad es el factor más determinante: los clientes jóvenes (25-30 años) tienen alta probabilidad de ser internautas (>0.80), mientras que los seniors (65-70 años) tienen baja probabilidad (<0.50). Para edades similares, la renta tiene un efecto moderado.

Un hallazgo interesante es que la renta alta no garantiza ser internauta si la edad es avanzada. El cliente senior con renta alta sigue siendo clasificado como no internauta, lo que refuerza la importancia dominante de la edad.
